## B. Пишем новое (этого в CodeFlow нет)

Приоритет сверху вниз.

- **Решение (2026-07-18): ElevenLabs — основной TTS, Fish — фоллбэк.** Fish НЕ удаляем — остаётся как запасной провайдер на случай сбоя/лимитов EL.

- Собранные входные данные: `voice_id = CICpbs1ZGqlhQNbQmCUP`, модель `eleven_turbo_v2_5` (turbo = мин. латентность для barge-in). Ключ (промо-подписка) лежит в `nation/.env`.
- ⚠️ Гэп перед свапом: движок читает `agentx/.env` (там сейчас **другой**, старый EL-ключ для STT-бейкоффа), а поля `elevenlabs_api_key` в `synapse/config.py` **нет**. Перед свапом: перенести ключ+voice_id в `agentx/.env` → добавить поле в `config.py` → поправить `validate_voice_keys()`.
- Рассмотреть **ElevenLabs Agents Platform** для системного промпта/инструментов вместо самодельного cascade — см. §D.3 и research R0.3/D3 (дефолт — свой cascade + EL TTS).

### B2. Telephony (исходящие звонки) — *обязательно*
CodeFlow — WebRTC/браузер (`pipeline/webrtc_server.py`). Negotiator нужен исходящий на реальные бизнесы. Варианты (бриф допускает все три):
- **Twilio/SIP** живые звонки (pipecat имеет транспорт; ElevenLabs интегрируется с Twilio).
- **Human-in-the-loop** — сам отвечаешь в разных ролях.
- **Симулированный рынок** — свои counter-агенты (agent-to-agent).
- **Рекомендация:** симулированный рынок для ≥3 стилей (надёжно на сцене) **+ один живой Twilio-звонок** для достоверности.

### B3. Estimator: путь документов — *обязательно*
Бриф требует **оба** пути к одному job spec: голос **И** разбор документов (фото, старые котировки, счета, описи).
CodeFlow делает только голосовой интейк (стадия COLLECT). Пишем: ingestion документа → та же JSON-схема job spec → та же стадия CONFIRM.

### B4. Конфиг-пак вертикали — *обязательно, как конфиг не код*
- Таксономия job spec (для переездов: объём, этажи/лифт, дистанция, дата, спецпредметы…).
- Ценовые бенчмарки (FMCSA, moveBuddha).
- Red-flag правила (−30% от рынка = тревога; sight-unseen).
- Рычаги переговоров и калиброванные вопросы (Getting to Yes: BATNA/интересы; Voss: калиброванные вопросы).
- Смена вертикали = подмена конфига (проверяемое требование брифа).

### B5. Multi-call сравнение + ранжированный отчёт — *обязательно*
Захват разбитых по статьям котировок из ≥3 звонков → нормализация в сравнимую форму → ранжирование → отчёт со ссылками на **транскрипты И аудио-записи** (`recording_url#t=sec`, Media Fragments URI; speaker бесплатно из Twilio dual-channel) + **плейн-текст рекомендация** → детекция red-flag. Опирается на `journal.py`, но сама логика сравнения/ранга — новая. (Закрывает дыру E брифа — «cite recordings AND transcripts».)

### B6. Counter-агенты (3 стиля) — *обязательно для демо*
Канон ролей (spec §3.10): `rushed_dispatcher` (перебивает, прячет сборы), `pressure_closer` (дедлайн + якорь,
уступает под рычагом), `lowball_broker` (−30% + sight-unseen, broker не carrier). Покрывают barge-in,
измеримый сдвиг цены из-за рычага и red-flag в разговоре — ровно демо-моменты §8.

### B7. AI-раскрытие и линия честности — *дизайн разговора (это и есть выигрышная ось)*
- Агент раскрывает, что он AI, действующий по поручению клиента; изящно обрабатывает «я говорю с роботом?».
- «Измеримое изменение цены из-за рычага, который агент **сам добыл**» — минимум один такой разговор.
- Технический предел блефа — через гейт B (adapt), не через промпт.

### B8. Discovery списка обзвона — *обязательно (бриф требует показать источник)*
Список бизнесов строится **программно**, не из конфига: **Google Places (New) Text Search**
(`includedType=moving_company`, field-mask `places.nationalPhoneNumber` — name+phone в одном ответе;
ключ self-serve GCP <10 мин, SKU ~$35/1k = для демо копейки). FMCSA — слой легитимности через `verify`
(ищет по имени/USDOT, не гео), не discovery. Yelp дисквалифицирован ($299/мес + триал запрещает деплой),
OSM/Overpass — бесплатный фолбэк с рваным покрытием. spec §3.8. (Закрывает дыру C брифа.)
